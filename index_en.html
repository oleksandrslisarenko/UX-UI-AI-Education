How Users Think

  • 8.1How to Write Prompts for AI Prototyping

    A mobile prototype requires a different prompt: you must explicitly specify the platform (iOS/Android), navigation type (tab bar / bottom nav), and touch element sizes. Without this, AI generates a web interface with a smaller font.

  • 11.1How to Present a Prototype to a Client: What to Say and What Not to Say
    A prototype is a question, not an answer. The right technique: not "here's what we built," but "help us validate this." For mobile — always show on a real device.

🎯 Final Project — Group Defense
Full stakeholder cycle:

1. Take a real or fictional client request (1 paragraph of requirements)
2. Build a user journey map
3. Choose an AI tool and generate a prototype (3–5 screens) — web or mobile
4. Conduct a heuristic audit using the NN/g workbook
5. Prepare a presentation scenario: 3 questions you want to validate with the client
6. Defend before the group (5 min) — speaker and participants give feedback

Evaluation: is there a clear user flow, does the prototype meet requirements, did the audit reveal real problems.
Course Outcome
Completed the full cycle: requirements → journey → prototype (web or mobile) → heuristic audit → client validation. I have practical tools and an understanding of quality UX principles.
